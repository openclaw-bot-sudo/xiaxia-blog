# 都是"做虾"，Claude 版凭什么比 OpenClaw 更香？

## 正文

▎Agent 潮起，生态之争不会缺席，"跟跑者"将很快遭遇自己的极限。

作者 | 胡珈萌  
编辑 | 杨林  
本文首发于钛媒体 APP

2026 年，AI 正在集体"虾化"。

从硅谷的 OpenAI、Anthropic，到国内大厂腾讯、阿里、字节，乃至华为、小米，都已亲自下场，动手做"虾"。

但在风起之处，"原版龙虾"OpenClaw 却遭遇了一次升级事故，也引发了人们对这一热潮最新的讨论和思考。

当地时间 3 月 22 日，OpenClaw 开发者彼得·斯坦伯格（Peter Steinberger）发帖宣布升级后的 OpenClaw v2026.3.22 预览版正式上线。

此次升级将底层模型更新到了 GPT-5.4，明确了 ClawHub 为首选分发渠道，并宣布彻底废弃旧有的扩展 API，代之以全新的模块化 SDK。同时，对于最受关注的安全问题，新版本也做了大幅调整，通过拦截 SMB 凭证泄露、环境隔离加固等方式修复了 10 余项漏洞，防范安全事故。

但不成想，这次重大升级自身反而成了重大事故。

大量进行了升级的用户反馈，根本无法加载控制界面，升级即趴窝。

国内包括微信 Clawbot 等"国产龙虾"也出现大面积报错。让不少刚刚习惯每日与"龙虾"交流的"养虾人"措手不及。

当地时间 3 月 23 日，斯坦伯格发帖解释称，v3.22 版本无法正确加载是因为自己"在发布 Web 控件 UI 资源时漏掉了一个步骤"。他也随即给出了修复版，并快速上线 v3.23 版。

不少网友吐槽，称"这是史诗级的马虎大意""确实是 Vibe Coding，也太草台了"，还有网友调侃斯坦伯格入职 OpenAI 后太忙，以至于犯下这种低级错误。

也有声音表示理解。长期关注 OpenClaw 动态的 AI 领域创业者肖炜对作者表示，官方升级本就是快速迭代和快速修复的模式。而且，从提升安全性的角度考虑，这种迭代和修复模式在目前阶段是必要的。

他也同时表示，鉴于 OpenClaw 目前主要仍是以斯坦伯格一人为主的维护模式，但其影响力和生态已经达到了世界级，如何从一个社区项目变成一个成熟的产品，仍有很多事情要考虑。他还建议使用原版 OpenClaw 的用户无需赶着更新到最新版，称"毕竟本身软件升级就是一个权衡利弊的事情"。

不过，值得注意的是，虽然 v3.23 解决了 OpenClaw 界面无法加载的问题，但更新后的诸多争议仍未平息。

这些争议主要集中在此次更新为安全考量，大举影响了 OpenClaw 的兼容性。

比如，上文提到的 SDK 取代旧有的扩展 API，并引入严格的沙箱隔离机制，让数以千计的第三方插件一夜之间失效，许多正常使用新版本的用户也抱怨升级后"插件全废""白忙活"。

此外，微信 ClawBot 也因属于插件接入，在升级后无法接入访问，至今尚未恢复。腾讯公关总监张军此前表示，微信很快会更新插件解决相关问题。

值得一提的是，"国产龙虾"报错也将讨论话题导向了"生态玩法"上。

社媒 X 上的知名科技账号@林亦 LYi 认为相关事件显示出"微信适配太心急"，并表示"如果想把龙虾肉吃好，微信得好好学习开源生态的玩法了"。

而作为"微信非官方发言人"的微信员工@客村小蒋则将之定义为"一个新产品的迭代里有点小问题"，并回应称"说微信不懂生态玩法，我觉得有点像说鱼不会游泳了。"

从相关评论区的讨论也可看出，各自观点均有支持者和反对者，有人认为 OpenClaw 愈发封闭、忘记"初心"才是问题所在，也有声音称如果国产 AI 只是跟跑、跟风，那就只能处于被动地位。

一位精通 OpenClaw 等 Agent 工具的开发者提示作者，此次版本更新的真正意义就在于，作为"开源精神""游戏精神"代表的 OpenClaw，也在巨大的 Agent 浪潮和愿景下，试图将生态自主权攥在自己手中。

而对于字节、腾讯这样掌握"超级入口"的国内大厂来说，过去它们是流量的中心，话语权在自己手里。但在脱胎于 OpenClaw 的"龙虾生态"中，它们更多精力集中在适配上，虽然接连推出了爆款产品、吸引了新一波流量，但话语权始终不掌握在自己手中。

上述开发者还预计，虽然目前引起了很大争议，但 OpenClaw 应该还是会变成"越来越封闭的开源项目"，无论是为了安全、产品还是生态自主权，这都是大概率事件。

对于真正有志于发展 Agent 的国内厂商来说，在抢到了这波"龙虾"流量和红利后，或许是时候认真思考如何"掌控自己的命运"了。

而在五花八门的众多"龙虾"中，或许 Anthropic 的"Claude 版龙虾"才是更值得学习的那一个。

Anthropic 刚刚推出的这款产品，一改 Claude 系产品此前在本地交互上相对保守的风格，不仅能够像 OpenClaw 一样操作本地电脑完成各类事务，在安全、服务、低门槛、生态整合方面也因其"官方背景"更具优势。

很多人或许已然忘记，OpenClaw 的灵感和关键推动力正来自 Claude，也因 Anthropic 的警告数次更名。

虽然说 Claude 版龙虾"绝杀了 OpenClaw"有些夸张，但其根据开源爆款重新调整产品策略，依托自身技术实力推出优化版，确实是让"龙虾"归自己所有所用，并实现持续进化，而非简单"跟跑"的更好方式。

上述开发者也提到，Claude 仅今年以来就发布了 20 多项新功能，"这个迭代速度从没看到有哪家公司能做到"。而这成为了 Anthropic 收入和月活大增的重要基础，也是其在 Agent 浪潮中取得先机，叫板 OpenAI 的底气所在。

Agent 时代才刚刚启幕，国产厂商想要养好自己的"龙虾"，那就不能只盯着"意外爆款"OpenClaw，至少也要向 Claude 的方向努努力。
